# 📝 How We Create Feature Breakdowns

## 📁 Documentation Structure
Feature breakdowns should be placed in the appropriate docs folder:

```
project/
  └── docs/
      └── feature_breakdowns/  # Feature implementations
```

## 📖 Required Document Structure

Each document has only these three sections:

### 1. Introduction Section
- Title: "# 📝 Introduction"
- Content: Explain what feature was implemented and why
- Include any relevant context or dependencies
- Keep it focused on the specific implementation

### 2. The Plan Section
- Title: "# 🎯 The Plan"
- Format: Numbered list
- Each step should be one sentence max
- Steps should reflect exactly what was done in order

### 3. Implementation Section
- Title: "# 💻 Implementation"
- Sub-sections should match each step from The Plan
- Include actual code snippets used
- Explain relevant concepts and patterns
- Highlight important decisions and why they were made

## ✍️ Style Guidelines
- Use emoticons for all main headers
- Keep explanations clear and practical
- Include actual code snippets used
- Use markdown formatting consistently
- Focus on the specific implementation details
- Explain why certain approaches were chosen

## 🚀 Best Practices
- Document immediately after implementation
- Include all relevant code changes
- Explain patterns that can be reused
- Link to related documentation
- Keep focus on the specific feature
